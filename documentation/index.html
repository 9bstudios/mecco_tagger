About Tagger

Tagger is a collection of tools for managing polygon tags and material assignments in MODO.

What are pTags?

Materials in MODO are assigned using "polygon tags" (pTags). Each polygon in a mesh is assigned a little chunk of text that--put simplisticly--tell MODO what material mask to use when shading said polygon.

Assigning materials this way is very flexible, because each polygon in a mesh can have its own material assignment. It's also inherently heavy and slow, since each and every polygon in the mesh has a little string of text attached to it. For very dense meshes (like CAD imports), this can become prohibitively heavy.

Types of pTags

There are three kinds of polygon tags in MODO: material tags, part tags, and pick tags (aka selection sets).

Material tags are typically used for material filtering in the shader tree. A given poly may only have one Material tag at a time.

Pick tags are the internal name for "selection sets", allowing the user to quickly save and recall polygon selections. Pick tags are unique because each tag is actually a semi-colon delimited list of individual tags, allowing a given polygon to have as many pick tags as needed. Since a polygon can have any number of pick tags, they can be very useful for complex layered material filtering.

Part tags are commonly used for storing extra information about polygons, especially during import from a third-party format. In CAD imports, for example, Part tags are used to demark individual CAD faces from the original NURBS data. Most users are unaware this data exists, but it can be invaluable when working with CAD data. A given polygon may only have one Part tag at a time.

Tagger Commands

  • Convert Tag Type

    tagger.pTagConvert

    Change a Tag from one type to another, e.g. convert a part tag to a material tag

    Typical Usecase

    This is an advanced tool for highly specialized applications.

    It is especially valuable when working with imported data, such as CAD data. Material tags imported from CAD may be useful for making selections, but less so for assigning materials. In that case, we can convert Material tags to Selection Sets, thus allowing us to assign new Material tags while maintaining access to the original imported material tags for reference. Or, in other cases, it may be useful to convert Part tags to Material tags or vise-verse, depending on the specifics of the CAD import process.

    Example

    1. select polygons
    2. run command
    3. select tag type to convert from
    4. select tag type to convert to
    5. hit enter

    Result: Tags on selected polys are converted.

  • Set Material (auto)

    tagger.setMaterial_auto

    Set material for polygons if in poly selection mode, or items if in item mode

    Simply selects the appropriate tagger.setMaterial command based on selection mode.

    • Item Mode: tagger.setMaterial_item.
    • Component Modes: tagger.setMaterial_pTag.

  • Remove Material (auto)

    tagger.setMaterial_autoRemove

    Remove material masks and tags from polygons if in poly selection mode, or item masks if in item mode

    Simply selects the appropriate material removal command based on selection mode.

    • Item Mode: tagger.setMaterial_itemRemove.
    • Component Modes: tagger.setMaterial_pTagRemove.

  • Set Material (Tag)

    tagger.setMaterial_Tag

    Set material for Tags (materials, parts, or selection sets)

    Probably the most common way MODO users assign materials (mapped to M by default).

    Example

    1. select polygons
    2. hit M for Material
    3. type a tag name (or select an existing one from the popup list)
    4. hit enter

    Result: Polys are tagged and, if appropriate, a shader tree mask is created.

    Arguments

    • Quick Preset
    • Scope
    • Tag Type
    • With Existing
