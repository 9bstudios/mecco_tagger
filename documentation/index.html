About Tagger

Tagger is a collection of tools for managing polygon tags and material assignments in MODO.

What are pTags?

Materials in MODO are assigned using "polygon tags" (pTags). Each polygon in a mesh is assigned a little chunk of text that--put simplisticly--tell MODO what material mask to use when shading said polygon.

Assigning materials this way is very flexible, because each polygon in a mesh can have its own material assignment. It's also inherently heavy and slow, since each and every polygon in the mesh has a little string of text attached to it. For very dense meshes (like CAD imports), this can become prohibitively heavy.

Types of pTags

There are three kinds of polygon tags in MODO: material tags, part tags, and pick tags (aka selection sets).

Material tags are typically used for material filtering in the shader tree. A given poly may only have one Material tag at a time.

Pick tags are the internal name for "selection sets", allowing the user to quickly save and recall polygon selections. Pick tags are unique because each tag is actually a semi-colon delimited list of individual tags, allowing a given polygon to have as many pick tags as needed. Since a polygon can have any number of pick tags, they can be very useful for complex layered material filtering.

Part tags are commonly used for storing extra information about polygons, especially during import from a third-party format. In CAD imports, for example, Part tags are used to demark individual CAD faces from the original NURBS data. Most users are unaware this data exists, but it can be invaluable when working with CAD data. A given polygon may only have one Part tag at a time.

Tagger Commands

  • Convert Tag Type

    tagger.pTagConvert

    Change a Tag from one type to another, e.g. convert a part tag to a material tag

    Typical Usecase

    This is an advanced tool for highly specialized applications.

    It is especially valuable when working with imported data, such as CAD data. Material tags imported from CAD may be useful for making selections, but less so for assigning materials. In that case, we can convert Material tags to Selection Sets, thus allowing us to assign new Material tags while maintaining access to the original imported material tags for reference. Or, in other cases, it may be useful to convert Part tags to Material tags or vise-verse, depending on the specifics of the CAD import process.

    Example

    1. select polygons
    2. run command
    3. select tag type to convert from
    4. select tag type to convert to
    5. hit enter

    Result: Tags on selected polys are converted.

  • Set Material (auto)

    tagger.setMaterial_auto

    Set material for polygons if in poly selection mode, or items if in item mode

    Simply selects the appropriate tagger.setMaterial command based on selection mode.

    • Item Mode: tagger.setMaterial_item.
    • Component Modes: tagger.setMaterial_pTag.

  • Remove Material (auto)

    tagger.setMaterial_autoRemove

    Remove material masks and tags from polygons if in poly selection mode, or item masks if in item mode

    Simply selects the appropriate material removal command based on selection mode.

    • Item Mode: tagger.setMaterial_itemRemove.
    • Component Modes: tagger.setMaterial_pTagRemove.

  • Set Material (Tag)

    tagger.setMaterial_Tag

    Set material for Tags (materials, parts, or selection sets)

    Probably the most common way MODO users assign materials (mapped to M by default).

    Example

    1. select polygons
    2. hit M for Material
    3. type a tag name (or select an existing one from the popup list)
    4. hit enter

    Result: Polys are tagged and, if appropriate, a shader tree mask is created.

    Arguments

    • Tag - The tag to apply. Type a new one or select an existing one from the popup.
    • Quick Preset - The material preset to apply.
      • "Random Color" applies a basic material with a semi-random Diffuse Color value.
      • All other entries are pulled dynamically from locations on disk. A collection of "Basic" materials (by Richard Yot) is included by default, and certain plugins (like Kelvin) will add themselves automatically to the list. You can add your own presets using Preferences > Mechanical Color > Tagger > User Presets Path.
    • Scope - By default, the tag will be applied to the selected polygons. For some workflows, however, it can be helpful to set the scope to Flood or Connected to save precious clicks.
      • Selected: Assign the tag to the selected polygons.
      • Flood: Assign the tag to contiguous patches of polygons with the same tag.
      • Connected: Assign the tag to the entire polygon island containing each polygon.
    • Tag Type - Most people use Material tags to assign material masks, but in some cases it can be helpful to filter using Part or Selection Set tags. Selection Sets are especially helpful for complex layered materials, since any given poly can have as many Selection Set tags as needed (instead of just one material tag). Tagger supports any tag type.
    • With Existing - The default M key behavior in MODO is to assign a tag first and, if no mask currently exists for said tag, create a mask for it. If a material mask already exists, however, it will simply use that one.
      • Use: This behaves exactly like the MODO default: if a mask exists, use it. If not, create one.
      • Keep and Add: Always adds a new mask to the top of the shader tree, even if another one already exists for the current tag.
      • Remove and Add: Remove all other masks in the shader tree that use the current tag, then add a new one to the top of the list.
      • Consolidate and Add: "The non-committal option." Collects all existing masks for a given tag into a single parent group mask, then adds a new one to the top of that list.

  • Set Material (Tag Islands)

    tagger.setMaterial_TagIslands

    Set a different material (random color) for each poly island in the active mesh item(s)

    Assigns a different random color to each polygon island in the active mesh.

    Arguments

    • Tag - The basename for the tags to be applied. Each polygon island will have this tag applied with a serial number appended to the end, e.g. "tag_1", "tag_2", etc.
    • Tag Type - Most people use Material tags to assign material masks, but in some cases it can be helpful to filter using Part or Selection Set tags. Selection Sets are especially helpful for complex layered materials, since any given poly can have as many Selection Set tags as needed (instead of just one material tag). Tagger supports any tag type.

  • Remove Material (Tag)

    tagger.removeMaterial_Tag

    Remove material Tags and masks for selected polygons

    Completely removes material, part, or selection set tags from the selected geometry.

    Arguments

    • Tag Type - By default, the command removes material tags for the selected geometry. Set to 'part' or 'pick' to remove part or selection set tags respectively.
    • Scope - By default, the tag will be removed from the selected polygons. For some workflows, however, it can be helpful to set the scope to Flood or Connected to save precious clicks.
      • Selected: Remove the tag to the selected polygons.
      • Flood: Remove the tag to contiguous patches of polygons with the same tag.
      • Connected: Remove the tag to the entire polygon island containing each polygon.
    • Delete Unused - When enabled, the command will also delete associated shader tree masks if there are no polys remaining with the polygon tag being removed.

  • Set Material (Group)

    tagger.setMaterial_group

    Create a new group containing the selected items, then Set Material using the newly created group

    Creates a Group, adds the selected item(s) to said Group, and creates a shader tree mask for it.

    With complex CAD assemblies it can be faster to apply item and group masks than to apply material tags to millions upon millions of polygons. Item masks work well for individual items, and Groups allow you to mask groups of items all at once--without changing the structure of your items list. This is a powerful and under-used feature of MODO, and one that Tagger makes much easier.

    Arguments

    • Name - The name of the group to be created.
    • Quick Preset - The material preset to apply.
      • "Random Color" applies a basic material with a semi-random Diffuse Color value for each item mask.
      • All other entries are pulled dynamically from locations on disk. A collection of "Basic" materials (by Richard Yot) is included by default, and certain plugins (like Kelvin) will add themselves automatically to the list. You can add your own presets using Preferences > Mechanical Color > Tagger > User Presets Path.

    tagger.TagSet

    Arguments

    • Tag Type - material, part, or pick (i.e. Selection Set)
    • Tag - The tag to apply.
    • Scope - By default, the tag will be applied to the selected polygons. For some workflows, however, it can be helpful to set the scope to Flood or Connected to save precious clicks.
      • Selected: Assign the tag to the selected polygons.
      • Flood: Assign the tag to contiguous patches of polygons with the same tag.
      • Connected: Assign the tag to the entire polygon island containing each polygon.

  • Tag Copy

    tagger.pTagCopy

    Copy and paste Tags from one poly to another

    Copies all tags from the first selected polygon. Copied values can be retrieved using the various tagger.pTagPaste... commands.

    Allows for copy and paste from one poly selection to another. Very convenient for quickly assigning existing materials visually. Mapped to ctrl + shift + C by default.

  • Poly Selection (Commands List)

    tagger.pTagSelectionFCL

    Form Command List of Tags in currently selected polys

    This queriable command is the basis for the "Tag Inspector" in the main Tagger UI. It generates a list of all tags in the current polygon selection--assuming that there are not (a) too many polygons to scan quickly, and (b) there are not too many tags to list on screen.

    Each tag is displayed as a tagger.selectAllByMaterial, tagger.selectAllByPart,or tagger.selectAllBySelectionSet form button.

    To use, simply place tagger.pTagSelectionFCL ? in a form.

  • Consolidate Masks by Color

    tagger.shaderTree_consolidateByColor

    Combine all simple material masks with the same diffuse color value.

    Tries to combine materials of the same Diffuse Color into named groups.

    Imported CAD data often comes in with large numbers of material tags describing a comparitively small number of materials. These are often shown in MODO under a generic name (e.g. material 0.0 1.0 0.5), and are often highly redundant.

    This command operates on any mask that (a) masks a Material tag, (b) has only one child, and (c) that child is of type advancedMaterial (i.e. a normal MODO material item). It checks the Diffuse Color of the material and catalogs the material tag. Once it has generated a list of unique colors in the scene, it then goes back through the tree hunting for redundant masks, re-assigning their poly tags, and deleting the old masks.

    New masks are named using the nearest web color name equivalent, but the color value itself is unchanged.

    Note: Because it involves re-tagging virtually every polygon in the scene, this process can take a few minutes depending on the size of the file being scanned.
